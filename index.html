  • Finance Reconciliation

  • Business Transformation Software Development Strategy Data Engineering

Project Overview

Owned the development and maintenance of a finance reconciliation service, which aggregated all charging sessions from the company's electric vehicle fleet and reconciled them with the company's financial system.


The first iteration of this system was an end-to-end, scalable revenue reconciliation service, which was responsible for the creation of invoices in thier accounting software.


This system eventually was replaced by a data pipeline, which was built using a number of ETL's created using Domo, the companies Data Warehouse. The reason for this is that Xero was no longer able to handle the volume of invoices being created. The solution was to roll up the invoices into a single invoice per month, per charging station, which was then sent to Xero.

Key Achievements

  • Created a scalable revenue reconciliation service, which was responsible for the creation of invoices in thier accounting software.
  • Developed a data pipeline, which was built using a number of ETL's created using Domo, the companies Data Warehouse.
  • Owned a system which rolled up the invoices into a single invoice per month, per charging station, which was then sent to Xero.
